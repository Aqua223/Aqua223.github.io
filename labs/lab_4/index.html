Лабораторная работа №4

Тема: Классификация с применением Scikit-Learn

Гугл коллаб

Цель работы

  • Освоить методы бинарной классификации с использованием библиотеки Scikit-Learn.
  • Научиться оценивать качество моделей с помощью метрик Accuracy, Confusion Matrix, ROC-AUC.
  • Исследовать альтернативные алгоритмы классификации (SVM, kNN, Gradient Boosting).
  • Проанализировать современные подходы к кредитному скорингу.
  • Изучить способ интеграции обученной модели в веб-сервис.

Ход работы

  1. Загружены данные Kaggle "Give Me Some Credit".
  2. Выполнена предобработка: пропуски заполнены средними значениями обучающей выборки.
  3. Обучены базовые модели:
  4. Logistic Regression
  5. Random Forest
  6. Рассчитаны метрики Accuracy и Confusion Matrix.
  7. Построены ROC-кривые, вычислен ROC-AUC.
  8. Исследовано влияние порога классификации (0.3, 0.5, 0.7) на матрицу сопряжённости.

Самостоятельная работа

1. Исследование других моделей классификации

Были дополнительно обучены и оценены модели SVM, kNN и Gradient Boosting.

Результаты:

Модель ROC-AUC
Gradient Boosting 0.8644
Random Forest 0.8514
kNN 0.7058
Logistic Regression 0.6618
SVM 0.6610

Вывод: Gradient Boosting показал наилучший результат (ROC-AUC = 0.8644). SVM и Logistic Regression дали одинаково низкие показатели (~0.66), что объясняется линейным характером границы решений. kNN оказался на среднем уровне (0.7058). Порог 0.9 не достигнут ни одной моделью. С помощью LLM установлено, что причина — сильный дисбаланс классов (всего 6% дефолтов) и ограниченность признакового пространства (10 переменных). Без привлечения дополнительных данных или глубокого feature engineering дальнейшее улучшение затруднительно.

2. Анализ современных алгоритмов классификации

Изучены научные публикации и Kaggle-соревнования по кредитному скорингу. Выявлено: - Лидируют LightGBM, CatBoost, XGBoost — алгоритмы градиентного бустинга, устойчивые к дисбалансу классов. - Применяются ансамбли (стекинг) — объединение нескольких моделей разной природы. - Используется калибровка вероятностей через CalibratedClassifierCV для повышения точности. - Тонкая настройка гиперпараметров ведётся автоматически через Optuna / Hyperopt.

3. Интеграция модели в веб-сервис — пошаговый алгоритм (FastAPI)

  1. Сохранить обученную модель: joblib.dump(gb_model, 'model.pkl').
  2. Создать FastAPI-приложение в файле main.py.
  3. Описать Pydantic-модель для входных данных (10 признаков клиента).
  4. Создать POST-эндпоинт /predict:
  5. Принять JSON с признаками.
  6. Заполнить пропуски средними значениями из обучения.
  7. Отмасштабировать данные (если требуется).
  8. Вызвать model.predict_proba() и вернуть вероятность дефолта.
  9. Собрать Docker-образ с приложением.
  10. Развернуть на облачном сервере (AWS, Yandex Cloud, Railway и т.д.).

Выводы

  • Освоены методы бинарной классификации в Scikit-Learn и ключевые метрики качества.
  • Gradient Boosting показал наилучший результат среди всех рассмотренных моделей (ROC-AUC = 0.8644).
  • SVM и Logistic Regression дали наихудшие результаты на несбалансированных данных (~0.66).
  • Изучены современные алгоритмы кредитного скоринга и способ интеграции модели в веб-сервис через FastAPI.
